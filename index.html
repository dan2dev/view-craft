view-craft

A DOM library for people who peaked in the jQuery era and never looked back.

Yes, this is yet another JavaScript library. We're sorry. Or you're welcome? We're still figuring it out.

Quick Example

import 'view-craft';

let count = 0;

const counter = div(
  h1(() => `Count: ${count}`),
  button('Increment', on('click', () => {
    count++;
    update();
  }))
);

render(counter);

Installation

Usage

import 'view-craft';

// Now use div(), update(), on(), list(), when(), render(), etc. globally
let count = 0;
const app = div(
  h1(() => `Count: ${count}`),
  button('Click', on('click', () => { count++; update(); }))
);
render(app);

TypeScript Setup

Add to your tsconfig.json:

{
  "compilerOptions": {
    "types": ["view-craft/types"]
  }
}

Core Concepts

1. Explicit Updates

Unlike React or Vue, view-craft doesn't auto-detect changes. You call update() when ready:

let name = 'World';

// Mutate freely
name = 'Alice';
name = name.toUpperCase();

// Update once when ready
update();

2. Reactive Functions

Zero-arg functions become reactive:

let count = 0;

div(
  () => `Count: ${count}`,  // Updates when update() is called
  { title: () => `Current: ${count}` }  // Attributes too
)

3. Conditional Rendering

First matching condition wins:

when(() => user.isAdmin,
  div('Admin Panel')
).when(() => user.isLoggedIn,
  div('User Dashboard')
).else(
  div('Please log in')
)

4. List Synchronization

Lists use object identity to track items:

list(() => items, (item, index) =>
  div(() => `${index}: ${item.name}`)
)
